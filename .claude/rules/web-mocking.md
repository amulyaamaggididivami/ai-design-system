# MSW Mocking Rules

## Architecture

- MSW v2 Service Worker for browser-level network interception (dev-mode only)
- Worker file: `apps/web/public/mockServiceWorker.js` (auto-generated by MSW)
- Shared handler definitions: `shared/src/mocking/` — reusable across apps
- Web-specific wiring: `apps/web/src/mocks/` — browser worker setup
- Mock DB: `@mswjs/data` for relational mock data
- Fake data: `@faker-js/faker` v10 for realistic values

## Rules

- IMPORTANT: guard MSW init with `import.meta.env.DEV` — never include in production bundle
- IMPORTANT: do NOT use MSW Node interceptor — use direct jest mocks in tests
- Handler definitions go in `shared/src/mocking/` so both apps can reuse them
- Mock responses must be typed and follow real backend API shape
- `shared/src/mocking/` is a standalone folder — no imports from other shared folders
- Mocking barrel: `shared/src/mocking/index.ts` — separate entry point from main shared exports
- Import via `@people-parley/shared/mocking` — never via the main `@people-parley/shared` alias
